## 1. Threat 

- A threat is any potential cause of an unwanted incident that can harm an assets ( data, system, network, or organization )

- _examples_ :
    - Malware
    - Insider misuse
    - Natural disasters
    - Phishing campaigns
    - System misconfiguration exposure

---

## 2. Threat Actor

- A threat actor is the individual, group, or entity that carries out a threat

- _common types_ : 
    - Cyber criminals ( financial gains )
    - Hacktivists ( ideology )
    - Nation-states (espionage, warfare)
    - Insider threats (employees)

- _examples_ : 
    - A ransomware gang targeting hospitals is a threat actor.

---

## 3. Threat Vector (Attack Vector)

- A threat vector is the path or method used to deliver a threat

- The route of Attack

- _common threat vectors_ : 
    - phishing emails
    - malicious websites
    - USB devices
    - Open network ports
    - Weak passwords
    - software vulnerabilities

- _examples_ : 
    - A phishing email carrying a malicious link is the threat vector

---

## 4. Vulnerability

- A vulnerability is a weakness that can be exploited by a threat.

- _vulnerability can exist in_ :
    - Software (unpatched systems)
    - Hardware (firmware flaws)
    - Network (open ports)
    - People (lack of training)
    - Processes (poor access control)

- _examples_ :
    - Weak passwords
    - Outdated software
    - Misconfigured cloud storage
    - No multi-factor authentication

---

## 5. Attack

- An attack occurs when a threat actor actively exploits a vulnerability using a threat vector

- the key difference between a "Threat" and "Attack" is 
    - Threat = potential
    - Attack = execution

- _examples_ : 
    - Launcing a phishing campaigns
    - Exploiting an unpatcher server.
    - Injecting SQL commands
    - Deploying ransomware

---

## 6. Breach

- A breach is a successful attack that results in unauthorized access, disclosure, or loss of data
- Not every attack results in a breach but every breach starts with an attack

- _examples_ : 
    - Customer data stolen
    - Credentials leaked
    - Systems encrypted by ransomware
    - Intellectual property exfiltrated

---

## 7. Risk

- Risk is the likelihood that a threat will exploit a vulnerability and the impact it would cause.

- _Risk = Threat x Vulnerability X Impact_

- _examples_ : 
    - High risk: Internet-facing server with known vulnerabilities
    - Low risk: Isolated system with limited access

---

## APT (Advanced Persistent Threat)

- An APT (Advanced Persistent Threat) is a highly sophisticated, long-term cyberattack where an attacker secretly gains access to a system or network and stays there for an extended period to steal data or monitor activity

- __What does "APT" mean__
    - _Advanced_ -> uses complex techniques (custom malware, zero-data exploits, social engineering) 
    - _Persistent_ -> The attacker remains inside the system for months or even years
    - _Threat_ -> usually organized groups (often state-sponsored or highly skilled cybercriminals)

- __Main goal of an APT__
    - unlike regular cyberattacks that aim for quick damage, APTs focus on:
        - stealing sensitive data (government, military, corporate secrets)
        - surveillance
        - sabotage

- __how an APT attack works__
    1. _Initial Access_
        - social engineering (phishing email)
        - exploiting software vulnerability
    2. _Establish foothold_
        - install backdoors or malware
        - create hidden admin accounts
    3. _Escalate Privileges_
        - Gain higher-level access (admin/root)
    4. _Lateral movement_
        - Move across the network to find valuable data 
    5. _Data exfiltration_
        - steal and send data outside the organization
    6. _Maintain Persistence_
        - stay hidden to continue spying

- __Common target__
    - government agencies
    - defense organization
    - financial institution
    - healthcare systems
    - large corporations

- __Examples of known APT groups__
    - APT28 (russia)
    - APT29
    - Lazarus group (north korea)
    - Equation group (china)

- __How to Protect Against APTs__
    - Multi-factor authentication (MFA)
    - Network segmentation
    - Regular patching and updates
    - Employee security awareness training
    - Intrusion detection systems (IDS/IPS)
    - Zero Trust security model
