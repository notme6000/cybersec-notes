### active recon
- ### ping
	- command live tool
	- active recon
	- ping is used for sending request to a ip and finding if the ip is active
	- `ping <example.com>` OR `ping <ip>`
	- calculates for RTT - round trip time, packet loss
	- ping sents ICMP ECHO request to a domain or an ip and checks if an ICMP ECHO response is recieved
	- RTT  - the time for a ICMP request to go to the server and come back
	- ICMP PROTOCOL
		- ICMP is a network layer protocol 
		- __INTERNET CONTROL MESSAGE PROTOCOL__
	- ##### options 
		- `ping -c 4 <ip>` - send only 4 packets or ICMP ECHO requests
		- `ping -i 2 <ip>` - sends request every 2 seconds ( firewall block prevention )
	- #### disadvantage
		- log will be stored
		- continuos icmp request from one mechine can or will be blocked by the server firewall
- ### nmap
	- __NETWORK MAPPING__
	- scanning and enumeration
	- open source
	- command line tool
	- `nmap <ip>` - scans ip for open ports
	- #### options
		- `nmap -sT <ip>` - tcp scan ( completes a full tcp handshake )
		- `nmap -sS <ip>` - stealth scan ( the tcp handshake is no completed but syn and ack is sent and recieved )
		- `nmap -sV <ip>` - version scan ( shows the versions of the running services in the system )
		- `nmap -O <ip>` - OS scan ( find the OS of the target mechine )
		- `nmap -A <ip>` - aggresive scan
			- version scan
			- os scan
			- script scan
			- traceroute
			- `nmap -sn <ip>` - ping scan ( only shows if the host is up or down no port scan )
			- `nmap -sF <ip>` - fin scan ( connect end/no response )
			- `nmap -sN <ip>` - null scan ( sends null value (000) to the target mechine )
